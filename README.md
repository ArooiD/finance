Ключевая идея:
Приложение является некоторым подобием банка, выполняющие финансовые услуги клиентов. Система реализует архитекутру Zero-Trust, за счёт компонентов Keycloak, CA.

Задачи:
- Используя контейнер infra.ca, в котором заложено приложение step-ca, выдать на каждый сервис в Docker сертификат используя механизм Acme.
- Реализовать по умолчанию https
- Разедлить Docker сеть на разные сегменты в зависимости от связности сервисов
- Поднять авторизационного провайдера Keycloak и связать с каждым сервисом проверку авторизационного токена и при каждом запросе, его проброс
- Реализовать минимальный набор функций по каждому сервису
- ...
